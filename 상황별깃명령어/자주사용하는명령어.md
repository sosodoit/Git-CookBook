## 새로운 브랜치 생성하고
```bash
$ git checkout -b 브랜치명   
$ git push --set-upstream origin 브랜치명 # 원격 저장소에도 브랜치 만들었다고 PUSH
```

## 커밋하고 
개발(작업)하고 변경된 사항 커밋할 때
```bash
$ git checkout -b 브랜치명 # 내 브랜치에서 작업을 했어야 함
$ git add . # .은 모든 변경사항을 올린다. 작업 단위로 추가하고 커밋해주는게 좋다.
$ git commit -m "커밋메세지"
```

## 푸쉬하고
```bash
$ git push origin 브랜치명 # 내 브랜치에서 git push 해도 되지만 안전하게 하자 
```

## 브랜치 삭제하고
목표하던 작업이 완료된 브랜치는 삭제하고, 병합이 완료된 main에서 새 브랜치 생성해서 다음 작업을 이어나간다.<br><br>협업시 병합하는 일자를 정하고, 병합이 완료된 시점에 모두와 같은 시점에 새 브랜치를 생성하는게 깔끔하다.

```bash
$ git branch -d 브랜치명        # 로컬 브랜치 삭제
$ git push origin -d 브랜치명   # 원격 브랜치 삭제
```
