# 새 상황 문서 추가 방법

새로운 "상황별 명령어" 문서를 추가할 때 따라주세요.

## 1. 위치

- **시작·연동**: `docs/01-시작하기/`
- **일상 작업(커밋·푸시·PR)**: `docs/02-일상-작업/`
- **브랜치**: `docs/03-브랜치/`
- **문제 해결(되돌리기·충돌·경고 등)**: `docs/04-문제해결/`
- **GitHub 협업(이슈·템플릿 등)**: `docs/05-GitHub-협업/`

해당하는 카테고리 폴더에 새 파일을 추가하면, MkDocs 빌드 시 사이드바에 자동으로 나타납니다. **`mkdocs.yml`을 수정할 필요 없습니다.**

## 2. 파일명

- 한글-하이픈 형식. 예: `커밋-취소-다시-커밋.md`
- 공백·특수문자 없이 작성합니다.

## 3. 문서 구성 (권장)

1. **제목**: 한 줄로 상황 설명 (H1 한 개)
2. **상황**: 1~2문장으로 "언제 쓰는지" 간단히
3. **명령어**: 코드 블록(`bash`)에 명령어만 넣어 복사하기 쉽게
4. **참고**: 필요 시 주의사항·추가 링크

## 4. 로컬 미리보기

```bash
pip install -r requirements.txt
mkdocs serve
```

브라우저에서 `http://127.0.0.1:8000` 으로 확인할 수 있습니다.
