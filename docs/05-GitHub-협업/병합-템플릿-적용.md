# 병합 템플릿 적용하기

PR을 **Merge**할 때 생성되는 merge commit 메시지 형식을 저장소에서 고르는 방법입니다.

## 상황

- 병합 커밋 메시지를 **PR 제목만** 쓰거나 **제목+설명**으로 통일하고 싶을 때
- 팀에서 병합 커밋 형식을 한 가지로 맞추고 싶을 때

## 꼭 필요한 것

- **위치**: GitHub 저장소 **Settings → General** 아래 **Pull Requests** 섹션
- **동작**: "Allow merge commits" 체크 후, 그 옆 **드롭다운**에서 형식 선택

## 선택 가능한 형식

| 옵션 | 병합 시 나오는 메시지 예시 |
|------|----------------------------|
| **Default** | `Merge pull request #123 from 계정/브랜치명` |
| **Pull request title only** | PR 제목만 (예: `feat: 로그인 API 추가`) |
| **Pull request title and description** | PR 제목 + 본문 설명 |

- 팀에서 "병합 커밋에는 PR 제목만 쓰자"로 정했으면 **Pull request title only**를 선택하면 됩니다.

## 언제 바꾸면 좋을지

| 상황 | 선택 예시 |
|------|------------|
| 히스토리를 짧게 가져가고 PR 단위로만 보고 싶음 | **Pull request title only** (또는 Squash and merge 사용 검토) |
| 병합 시 PR 본문까지 커밋에 남기고 싶음 | **Pull request title and description** |
| 기본 GitHub 스타일 유지 | **Default** 그대로 사용 |

Squash merge / Rebase merge를 쓰는 경우, 같은 **Pull Requests** 설정에서 squash용 기본 메시지 형식(제목만 / 제목+커밋 목록 등)을 따로 고를 수 있습니다.
