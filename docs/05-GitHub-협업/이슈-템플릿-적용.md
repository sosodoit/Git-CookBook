# 이슈 템플릿 적용하기

**New Issue** 클릭 시 미리 정한 양식이 본문에 나오게 하는 방법입니다.

## 상황

- 이슈를 만들 때마다 "뭘 적지?" 없이 **항목이 채워진 폼**이 나오게 하고 싶을 때
- 버그 / 기능 제안처럼 **유형별로 다른 양식**을 쓰고 싶을 때

## 꼭 필요한 것

- **단일 템플릿**: `.github/ISSUE_TEMPLATE.md` 파일 하나
- **여러 템플릿**: `.github/ISSUE_TEMPLATE/` 폴더 + 그 안에 `.md` 파일 (템플릿당 1개)

---

## 예시 1: 단일 템플릿 (항목만 최소)

파일: `.github/ISSUE_TEMPLATE.md`

```markdown
## 작업 내용
(무엇을 할지 적어 주세요)

## 체크리스트
- [ ] 항목1
- [ ] 항목2
```

- New Issue 시 위가 그대로 본문에 채워집니다. YAML(`---` 블록) 없어도 동작합니다.

---

## 예시 2: 여러 템플릿 (버그 / 기능 두 가지만)

**버그 리포트** — `.github/ISSUE_TEMPLATE/bug_report.md`

```markdown
---
name: Bug report
title: '[Bug] '
labels: bug
---

## 설명
(무엇이 잘못되었는지)

## 재현 방법
1. 
2. 
3. 

## 예상 동작
(어떻게 되어야 하는지)
```

**기능 제안** — `.github/ISSUE_TEMPLATE/feature_request.md`

```markdown
---
name: Feature request
title: '[Feature] '
labels: enhancement
---

## 요청 내용
(어떤 기능이 필요한지)

## 이유
(필요한 이유를 간단히)
```

- New Issue 시 "Bug report", "Feature request" 중 하나를 고를 수 있습니다. `labels`는 저장소에 있는 라벨 이름과 맞추면 자동 지정됩니다.

**반영**

```bash
git add .github/ISSUE_TEMPLATE.md
# 또는
git add .github/ISSUE_TEMPLATE/
git commit -m "docs: 이슈 템플릿 추가"
git push origin main
```

## 언제 수정하면 좋을지

| 상황 | 수정 예시 |
|------|------------|
| 작업 유형이 늘어남 | `.github/ISSUE_TEMPLATE/`에 `docs.md`, `question.md` 등 추가 |
| 팀 체크리스트가 정해짐 | 단일 템플릿의 "체크리스트" 항목을 팀 규칙에 맞게 수정 |
| 라벨 이름이 바뀜 | YAML의 `labels:` 값을 저장소 라벨과 동일하게 수정 |
| 버그 리포트에 환경 정보 필요 | `bug_report.md`에 `## 환경 (OS, 브라우저 등)` 섹션 추가 |
