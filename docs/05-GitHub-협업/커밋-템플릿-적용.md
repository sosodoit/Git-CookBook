# 커밋 템플릿 적용하기

`git commit` 시 같은 형식(타입·제목·이슈 번호)이 기본으로 열리게 하는 방법입니다.

## 상황

- 팀에서 커밋 메시지 규칙을 쓰고 싶을 때
- 매번 빈 편집기 대신 안내가 있는 상태로 열리게 하고 싶을 때

## 꼭 필요한 것

1. **템플릿 파일** 하나 (예: `.gitmessage`)
2. **설정 한 번**: `git config commit.template .gitmessage`

## 템플릿 예시 (필수 항목만)

프로젝트 루트에 `.gitmessage` 파일을 만들고, 아래만 넣어도 충분합니다.

```text
# <type>: <제목> (50자 이내)
# type: feat, fix, docs, refactor, chore
#
# 이슈: #
```

- `#`로 시작하는 줄은 주석이라 커밋 메시지에 포함되지 않습니다. 채울 부분만 남기고 나머지는 지우면 됩니다.

**설정**

```bash
git config commit.template .gitmessage
```

이후 `git commit`만 하면 위 내용이 편집기에 채워진 상태로 열립니다.

## 언제 수정하면 좋을지

| 상황 | 수정 예시 |
|------|------------|
| 팀에서 type 규칙이 바뀜 | `.gitmessage`의 type 예시 줄 수정 (예: `style`, `test` 추가) |
| 본문을 꼭 쓰기로 함 | `# (선택) 본문: 변경 이유, 관련 이슈` 같은 줄 추가 |
| 전역으로 쓰고 싶음 | `git config --global commit.template "경로/.gitmessage"` 로 같은 파일 지정 |

## 참고

- 설정 확인: `git config commit.template`
- 전역 설정은 다른 저장소에도 적용됩니다. 프로젝트만 쓰려면 저장소 루트에 `.gitmessage` 두고 위처럼 `commit.template`만 설정하세요.
