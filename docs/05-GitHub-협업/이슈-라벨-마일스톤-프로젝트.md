# 이슈, 라벨, 마일스톤, 프로젝트

GitHub에서 작업을 정리하고 진행 상황을 관리할 때 쓰는 기능 요약입니다.

## 상황

- 할 일을 **이슈**로 정리하고, **라벨**로 유형·단계를 구분하고 싶을 때
- **마일스톤**으로 목표일을 잡고, **프로젝트 보드**로 전체 진척을 보고 싶을 때

---

## Issues

작업 단위를 하나의 "할 일"로 정의하는 곳입니다. 제목·설명·담당자·라벨·마일스톤을 붙일 수 있습니다.

**사용 흐름**

- 저장소 **Issues** 탭 → **New issue**로 생성
- **제목**: 무엇을 할지 한 줄로 (예: `[TASK1] 로그인 API 구현`)
- **본문**: 체크리스트, 상세 조건, 참고 링크 등을 마크다운으로 작성

**커밋·PR과 연결**

- 커밋 메시지나 PR 본문에 `#이슈번호`를 넣으면 해당 이슈에 자동으로 링크됩니다.

```bash
git commit -m "docs: 이슈 사용법 작성 #1"
```

- PR 본문에 `closes #3` 또는 `fixes #3`을 넣으면, 그 PR이 병합될 때 이슈 #3이 자동으로 **Close** 됩니다.
- 이슈 템플릿을 쓰면 [이슈 템플릿 적용하기](이슈-템플릿-적용.md)를 참고하세요.

---

## Labels

이슈(또는 PR)에 붙이는 **태그**입니다. 작업 단계, 유형, 우선순위 등을 구분할 때 씁니다.

**예시**

- 단계: `TASK1`, `TASK2`, `backend`, `frontend`
- 유형: `bug`, `documentation`, `enhancement`
- 우선순위: `high`, `low`

**설정**

- **Issues** 또는 **Pull requests** 목록 오른쪽에서 **Labels** 클릭 → **New label**로 이름·색상·설명 지정
- 이슈 템플릿의 YAML에 `labels: bug`처럼 넣어 두면, 해당 템플릿으로 만든 이슈에 라벨이 자동 지정됩니다.

---

## Milestones

여러 이슈를 묶어 **목표일(마감일)**을 정하고, 진행률을 한 번에 보는 단위입니다.

**사용 흐름**

- **Issues** 탭 → **Milestones** → **New milestone**
- 제목(예: `1차 배포`), 기한(Due date), 설명 입력 후 생성
- 이슈를 열거나 수정할 때 **Milestone** 드롭다운에서 해당 마일스톤을 선택해 연결

**효과**

- 마일스톤 페이지에서 "열린 이슈 / 닫힌 이슈" 개수와 진행률(%)을 볼 수 있습니다.
- 기한이 지나면 자동으로 "Overdue"로 표시됩니다.

---

## Projects

이슈·PR을 **보드(Kanban)**나 **테이블** 형태로 모아서 진행 상황을 관리하는 기능입니다.

**사용 흐름**

- 저장소 상단 **Projects** 탭 → **New project**
- **Template**에서 Basic Kanban 등 선택 시 "Todo / In Progress / Done" 같은 컬럼이 생깁니다.
- 보드에 **Add item**으로 이슈·PR을 추가하거나, 이슈/PR 쪽에서 **Projects**에 연결

**효과**

- 할 일·진행 중·완료를 한눈에 구분할 수 있습니다.
- 팀 단위로 쓰려면 **Organization**에서 프로젝트를 만들고 여러 저장소 이슈를 한 보드에 모을 수도 있습니다.

---

## 요약

| 기능 | 쓰는 목적 |
|------|------------|
| **Issues** | 할 일 단위 정의, 제목·본문·담당·라벨·마일스톤 지정, 커밋/PR과 `#번호`로 연결 |
| **Labels** | 이슈·PR을 유형·단계·우선순위로 구분하는 태그 |
| **Milestones** | 이슈 묶음에 목표일을 두고 진행률·마감 관리 |
| **Projects** | 이슈·PR을 Kanban/테이블로 모아 진척도 시각화 |
