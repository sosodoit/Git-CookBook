# 이슈, 라벨, 마일스톤, 프로젝트

GitHub에서 작업을 정리할 때 쓰는 기능 요약입니다.

- **Issues**: 작업 단계별 업무·서브업무 정의
- **Labels**: 태그처럼 이슈 구분
- **Milestones**: 개발 목표일(마감일) 지정
- **Projects**: 전반적인 진행 현황 관리

## Issues

- **제목**: 작업 단계별 업무 정의
- **내용**: 체크리스트 등으로 상세 작성

커밋 메시지 끝에 Issue 번호를 붙이면, 해당 커밋이 이슈에 자동으로 연결됩니다.

```bash
git commit -m "docs: 이슈 사용법 작성 #1"
```

## Labels

- 작업 단계(예: `TASK1`), 유형(예: `RECORD`) 등으로 이슈를 구분할 수 있습니다.
- 이슈가 많아졌을 때 어떤 작업에 대한 이슈인지, 유형을 나누는 데 유용합니다.

## Milestones

- 작업 단계별 목표일을 설정해, 해당 작업에 속한 이슈를 한꺼번에 관리할 수 있습니다.

## Projects

- 프로젝트를 추가하면 Basic Kanban 템플릿이 생성되고 Issue와 연결할 수 있습니다.
- 전반적인 작업 진척도를 한눈에 볼 수 있습니다.
