# 커밋·푸시·PR 요청

변경 사항을 커밋하고 원격에 푸시한 뒤, PR(Pull Request)을 요청하는 요약입니다.

## 상황

브랜치에서 작업을 마친 뒤, 원격 저장소에 반영하고 메인(또는 대상 브랜치)에 반영 요청을 할 때입니다.

## 커밋

```bash
git add .
git commit -m "커밋메시지"
```

## 푸시

```bash
git push origin 브랜치명
```

## PR 요청

1. GitHub(또는 사용 중인 플랫폼) 저장소 페이지로 이동
2. 푸시한 브랜치가 보이면 **Compare & pull request** 클릭
3. 제목·설명 작성 후 PR 생성

## 참고

- 커밋 메시지 끝에 이슈 번호를 붙이면 해당 이슈와 자동 연결됩니다. 예: `git commit -m "docs: 내용 수정 #1"`
