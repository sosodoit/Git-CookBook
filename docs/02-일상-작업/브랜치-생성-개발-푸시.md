# 브랜치 생성하고 개발·커밋·푸시하기

새 브랜치를 만들고, 작업한 뒤 커밋·푸시까지 하는 흐름입니다.

## 상황

기능 개발이나 수정을 위해 새 브랜치에서 작업을 시작할 때 사용합니다.

## 새 브랜치 생성

```bash
git checkout -b 브랜치명
git push --set-upstream origin 브랜치명
```

## 개발 후 커밋

작업한 뒤 변경 사항을 커밋할 때입니다. 반드시 해당 브랜치에서 실행하세요.

```bash
git add .
git commit -m "커밋메시지"
```

- `git add .`는 모든 변경 사항을 스테이징합니다. 작업 단위로 `git add 파일명` 후 커밋하는 것을 권장합니다.

## 푸시

```bash
git push origin 브랜치명
```

브랜치를 지정해 푸시하면 실수로 다른 브랜치에 올리는 일을 줄일 수 있습니다.

## 참고

- PR 요청은 GitHub/GitLab 등에서 해당 브랜치를 선택해 생성합니다.
- 브랜치 삭제는 [브랜치 확인·생성·삭제](../03-브랜치/브랜치-확인-생성-삭제.md)를 참고하세요.
