# 삭제한 브랜치 복구

실수로 삭제한 브랜치를 다시 살리고 싶을 때 사용합니다.

## 상황

`git branch -d 브랜치명`으로 로컬 브랜치를 삭제했는데 다시 필요해졌을 때, reflog로 해당 커밋을 찾아 복구할 수 있습니다.

## 명령어

삭제한 브랜치와 커밋 해시를 확인한 뒤, 그 해시로 새 브랜치를 만듭니다.

```bash
git reflog
git checkout -b 삭제한브랜치명 해시값
```

- `git reflog`에서 삭제한 브랜치가 있던 커밋의 해시값을 찾습니다.
- `git checkout -b 브랜치명 해시값`으로 그 시점을 가리키는 새 브랜치를 만듭니다.
